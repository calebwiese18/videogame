6) DESIGN SYSTEM SPEC (TOKENS + RULES, NOT VIBES)
═══════════════════════════════════════════════════════════════════════
Define tokens and rules BEFORE implementation:

TYPOGRAPHY (make it archetype-appropriate)
- Font stacks (UI + optional display)
- Scale (role → size → line-height → weight → usage)
- Rules for headings, body, labels, numerals
- Reading measure rules for text pages (target comfortable line length; adapt per breakpoint)

SPACING + LAYOUT
- Base unit (4 or 8)
- Spacing scale (named steps)
- Container widths (content, wide, full-bleed)
- Vertical rhythm rules per section type (hero, content section, table, form)
- Grid rules (columns, gutters) per breakpoint

COLOR (semantic tokens)
- Surface hierarchy (page/surface/elevated/overlay)
- Text hierarchy (primary/secondary/tertiary/disabled/inverse)
- Borders + dividers
- Brand/accent usage rules (limited, intentional)
- State colors (success/warn/error/info) with accessible contrast

RADIUS / ELEVATION
- Radius tokens (none/sm/md/lg) with usage rules
- Elevation tokens (0–3) and when each is allowed

INTERACTION + MOTION
- State styling (hover/focus/active/disabled)
- Focus ring spec (consistent)
- Transition timing (micro vs layout)
- Prefers-reduced-motion compliance

COMPONENT STANDARDS
- Buttons (hierarchy + sizing rules)
- Forms (labeling, helper text, error states)
- Tables/lists (alignment, density modes)
- Empty/loading states (designed, not default)

7) IMPLEMENTATION PLAN (PROPAGATE VIA SYSTEM LAYERS)
═══════════════════════════════════════════════════════════════════════
Sequence:
1) Global tokens (CSS variables / theme / Tailwind config)
2) Base styles (typography defaults, link styles, selection, focus)
3) Layout shell (nav/header/sidebar, page grid)
4) Primitives (Button/Input/Card/Badge/Table/Dialog)
5) Composite patterns (filters, pagination, settings panels, content headers)
6) Pages last (only what doesn’t fall out of shared system changes)

Rules:
- No magic numbers: if you use a number, name it (token).
- No one-off styling unless justified.
- Prefer composition/slots over 12 boolean props.
- Remove duplicated components; consolidate.
